# Review References Before Planning

Before you draft implementation plan, check task references and documentation. References usually point to code files, GitHub issues, or URLs tied to the task. Documentation usually points to design docs, API specs, or other context needed for understanding.

These fields are visible in task view output. Read them before you decide how to implement.

## Observations
- [workflow] Review task references before writing implementation plan
- [workflow] Review task documentation before writing implementation plan
- [fact] References usually mean related code files, issues, or URLs
- [fact] Documentation usually means design docs, API specs, or other explanatory material
- [impact] Planning is weaker when references and documentation are skipped
