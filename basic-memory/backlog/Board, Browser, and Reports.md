# Board, Browser, and Reports

Backlog.md is not only task-edit CLI. It also has terminal Kanban board, web browser UI, and reporting/export features. If you search for kanban board or browser UI, this is the right note.

Use `backlog board` for terminal board view. Use `backlog browser` for web UI. The source doc also says export and reporting features exist for markdown reports and board snapshots, though exact command syntax is not captured there.

## Observations
- [fact] `backlog board` provides terminal-based Kanban board view
- [fact] `backlog browser` provides web UI view
- [fact] Export and reporting features exist for markdown reports and board snapshots
- [usage] Board and browser are for visualization, not replacement for CLI mutation rules
